# WEB 前端网站综合设计（前端设计）

## 一、风格设计与代码规范
(1) 禁止使用任何 CDN 网络资源，如SVG、PNG、JPG等格式的图标，CSS 和 JS 的库;  
(2) 所有 HTML 文件放在静态资源目录的根目录下，所有 CSS 文件放在 静态资源目录的 css/ 子目录下，所有 Javascript 文件放在静态资源目录的 js/ 子目录下，所有 SVG 图标或 PNG 图标文件放在静态资源目录的 icon/ 子目录下，所有 JPG、PNG、WEBP、GIF、AVIF格式的插图文件都放在静态资源目录下的 img/ 子目录下;  
(3) HTML, CSS, JS 文件分离到三个同名（文件扩展名除外）的文件中，禁止放在同一个文件中，例如：`index.html`, `css/index.css`, `js/index.js` ;  
(4) 所有 HTML 文件均使用简体中文，UTF-8 编码格式;  
(5) 主题采用深色主题，主题色使用蓝色系和黑色系;  
(6) 菜单栏没有连接的均设置为空超级链接(URL可设置为“#”);  
(7) 所有控件美观大气的同时，要求排版紧凑，不要间隔太大，注意响应式设计，主要以电脑端网页为主;  
(8) 当鼠标悬停于菜单项的文字上时，文字背景色变色，达到炫酷效果;  


## 二、控件校验输入数据有效性
(1) 密码使用 6~18 位长度，同时存在数字、大写字母、小写字母、其他字符中的至少其中两种，输入时实时显示密码输入是否合法；  
(2) 电子邮箱地址有效性：有且只存在一个 @ 字符，@ 字符不为字符串头部或尾部，
@ 字符尾部必须是至少包含一个小数点的域名，且小数点不能位于子字符串的头部或尾部，如果有多个小数点，小数点不能紧挨着，邮箱地址限制长度为50位，例如 contact@email.com 为合法邮箱地址；  
(3) 性别提供包含“男”和“女”的单选互斥输入控件;  
(4) username用户名仅允许使用小写字母、数字和下划线，且不允许以数字为首，若输入大写字母输入控件自动变为小写字母，长度限制在4到8位；
